# NAS 公共记忆服务说明

这个项目现在按“新系统”来理解，不继承旧数据。

## 目标

把 `global_context_db` 做成一个可以放在 NAS 上长期运行的公共记忆服务，给 Codex、OpenClaw、浏览器插件、桌面管理器等工具统一访问。

## 核心定位

- 数据集中存放在 NAS
- 所有工具只访问一个后端 API
- 桌面端只是管理界面，不是数据中心
- Docker 是 NAS 部署方式，不是用户必须理解的产品本体

## 现在的正确方向

1. 一个统一记忆库
2. 一个统一采集入口
3. 一个统一搜索入口
4. 一个统一的工具访问协议

## 第一版先做什么

- 记忆的增删改查
- 文档导入和搜索
- 公开 URL 采集
- RSS 导入
- 批量 URL 导入
- 给 AI 工具用的 API Key

## 暂时不做什么

- 不做复杂账号体系
- 不做平台反爬绕过
- 不做多库分片
- 不做云盘实时数据库

## NAS 上的使用方式

推荐方式是：

`NAS + Docker + 后端 API + 数据卷`

外部工具通过 HTTP 或 MCP 访问，不直接操作数据库文件。
